# Agent Instructions

## Setup

After checkout, run:

```bash
make init
```

This installs local hooks and upstream skills from `agents.toml`.
Use `make skills` to refresh upstream skills and `make format` before finishing edits.

## Skill Structure

```
skills/<skill-name>/SKILL.md
skills/<skill-name>/SPEC.md
```

Canonical personal skills live in `skills/<skill-name>/`.
`.agents/` and `agents.lock` are generated by dotagents; do not edit them by hand.
`.claude/skills` points at dotagents-managed skills for local tool compatibility.

## Creating/Updating Skills

Use `skill-writer` from `getsentry/sentry-skills` for new skills and material updates.
It handles source coverage, authoring, registration, and validation.

- Do **not** create per-skill alias or symlink skills
- Do not add skill inventories to `README.md`; the repo tree is the inventory
- Do not add agent, tool, or AI attribution to generated work
- Remove bloat: keep only instructions that change agent behavior

## Key Conventions

- Frontmatter `---` must be the **first line** of SKILL.md — no comments before it
- `name` field must match the directory name exactly
- `description` includes trigger keywords — this is how agents discover the skill
- Python scripts: always use `uv run <script>`, never `python` or `python3`
- Keep SKILL.md under 500 lines; move reference material to `references/`
- Keep runtime instructions in `SKILL.md`; put intent, source/evidence model, evaluation, limitations, and maintenance rules in `SPEC.md`
